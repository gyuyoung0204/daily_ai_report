# 📊 품질 루브릭 (Quality Rubric) — 단일 출처(SSOT)

> 이 파일은 산출물(아침 AI 다이제스트)과 프로젝트의 품질을 채점하는 **유일한 기준표**입니다.
> 루브릭 수정은 이 파일만 고칩니다. 스킬(`/quality-check`)·에이전트(`quality-evaluator`)는 참조만 합니다.
> 설계 근거: [docs/superpowers/specs/2026-06-12-quality-check-skill-design.md](docs/superpowers/specs/2026-06-12-quality-check-skill-design.md)

---

## 1. 채점 방식 (이중 평가)

각 차원을 **0~4점**으로 채점하되 두 종류로 나눠 산출한 뒤 종합한다.

- **정형평가(Quantitative):** grep·파일 존재·이슈 상태·테스트 결과 등 **기계적으로 확인 가능한 증거**. 같은 입력이면 누가 돌려도 같은 점수.
- **정성평가(Qualitative):** 지표로 안 잡히는 부분을 에이전트가 읽고 **근거를 인용**하며 채점.

두 점수가 크게 벌어지면(예: 정형 高 / 정성 低) 그 **불일치를 리포트에 명시**하고 개선 우선순위 근거로 쓴다.

**총점 공식:** `Σ(차원점수 × 가중치) / 4 × 100`
**등급:** 90–100 🥇 Production / 75–89 🥈 Solid / 60–74 🥉 Working / <60 🚧 Prototype

---

## 2. 평가 차원 (7종) · 가중치

| # | 차원 | 가중치 | 주 평가축 |
|---|------|--------|-----------|
| 1 | 기능 완성도 | 25% | 정형 |
| 2 | 데이터 신뢰성·정확성 | 20% | 정성 |
| 3 | 견고성·에러 처리 | 15% | 정형 |
| 4 | 테스트·검증 | 15% | 정형 |
| 5 | 자동화·운영 | 10% | 정형 |
| 6 | 문서 품질 | 10% | 정성 |
| 7 | 보안 | 5% | 정형 |

## 3. 차원별 Lv0~4 기준

### 1. 기능 완성도 (25%, 정형)
- **정형 지표:** 다이제스트 HTML 정상 생성(exit 0) / `$rawNews`가 실데이터인가 / 실 API·RSS 호출 코드 유무 / 인사이트가 수집 뉴스와 연동되는가(하드코딩 아님)
- Lv0 생성 실패 · Lv1 생성되나 데이터 하드코딩 · Lv2 실데이터지만 수집 수동 · Lv3 수집 반자동(WebSearch) · Lv4 수집·생성·인사이트 전 과정 자동·연동

### 2. 데이터 신뢰성·정확성 (20%, 정성)
- **정성 기준:** 출처 신뢰도, 기사 선정의 시의성, 사실 왜곡(환각) 없음, 도메인 관련성
- **정형 보조:** 기사별 출처 링크 표기율(목표 100%), TF-IDF 중복 제거 동작, 타겟 무관 기사 건수(↓)
- Lv0 출처 불명/환각 · Lv2 출처 있으나 일부 무관·구식 · Lv4 전 기사 신뢰 출처·시의적절·무관 0건

### 3. 견고성·에러 처리 (15%, 정형)
- **정형 지표:** `try/catch` 존재, 하드코딩 절대경로 개수(`C:\tmpfile` grep = 0 목표), 종료코드 처리, 템플릿/파일 부재 시 graceful fail
- Lv0 에러 처리 없음 · Lv2 부분 처리 · Lv4 전 경로 상대화·예외 처리·종료코드 정상

### 4. 테스트·검증 (15%, 정형)
- **정형 지표:** `test_*.ps1` 수 및 통과 여부, 검증 서브에이전트(quality-evaluator) 존재, 검증 훅(verify-digest) 배선
- Lv0 테스트 없음 · Lv2 일부 모듈 테스트 · Lv3 검증 에이전트/훅 존재 · Lv4 생성→자동 검증 컨베이어 동작

### 5. 자동화·운영 (10%, 정형)
- **정형 지표:** 스케줄 등록 여부, `logs/metrics.json`·`aggregated_summary.json` 존재, 수동 개입 횟수(↓, 한 번 트리거 목표)
- Lv0 전부 수동 · Lv2 생성만 자동 · Lv4 수집~검증 한 번 트리거·스케줄링

### 6. 문서 품질 (10%, 정성)
- **정성 기준:** 3대 문서(SOUL/CLAUDE/README) 중복 없음(SSOT), 코드와 일치, 진행률 정확성, 최신성
- Lv0 문서 부재/불일치 · Lv2 존재하나 일부 stale · Lv4 역할 분리·중복 0·사실 일치

### 7. 보안 (5%, 정형)
- **정형 지표:** 토큰 `.gitignore` 등록, 하드코딩 자격증명 grep 적발 수(0 목표), 토큰 커밋/로그 노출 없음
- Lv0 토큰 노출 · Lv2 부분 보호 · Lv4 토큰 gitignore·노출 0건

## 4. 좋은 산출물의 출력 지표 (참고, 차원 2·6에 반영)

| 지표 | 측정 | 좋은 방향 |
|------|------|-----------|
| 수동 개입 횟수 | 한 번 트리거 시 사람 개입 횟수 | ↓ (0회) |
| 중복/무관 유입 | 중복 클러스터·무관 기사 건수 | ↓ (0건) |
| 출처 표기율 | 기사별 출처 링크 비율 | ↑ (100%) |
| 도메인 맥락 연계 | 우리 업무 연관성 해석 수준 | ↑ (5점) |
| 가독성·충실도 | 핵심 전달 수준(1~5) | ↑ (5점) |

---

## 5. 자가학습 연계

채점 후 가장 낮은 가중점수 차원이 **다음에 고칠 곳**이다. 회차별 점수는 [logs/quality_scores.json](logs/quality_scores.json)에, 교훈은 [logs/lessons.md](logs/lessons.md)에 누적한다. 다음 채점 전 `lessons.md`를 먼저 읽어 같은 결함을 반복하지 않는다.
